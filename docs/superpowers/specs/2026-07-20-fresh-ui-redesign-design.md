# Akeso 清新 UI 全链路改版设计

日期：2026-07-20  
状态：已确认视觉方向，待实施

## 目标

在不改动 Akeso 现有业务状态、路由和领域数据的前提下，重做 Welcome、Check-in、Today、Plan 与 Nutrition 的界面。新版本应当清新、有识别度、亲近但不幼稚，并摆脱常见 AI 健康应用的白卡片、蓝绿渐变和平均化信息层级。

## 核心视觉方向：清醒能量伙伴

- 使用奶油白背景、墨黑文字、春芽绿主色，并以湖蓝、酸柠黄和柔和珊瑚色区分能量和营养状态。
- 版式参考独立杂志与生活方式应用：大字号标题、留白、局部高对比、错位叠层和少量手写感注释。
- Akeso 拥有专属“能量伙伴”角色。角色及装饰图像使用内置 Image-to-Image 生图能力，参考用户提供的界面图生成，不以 SVG 手工绘制角色。
- 角色在高、中、低能量与完成任务等状态中保持同一视觉身份，只调整表情、动作和局部颜色。

## 卡片与构图语法

界面不使用单一、重复的大圆角卡片系统，而采用“规则信息 + 自由焦点”的混合语法：

- 需要快速扫读和比较的数据，例如能量因子、营养数值和任务列表，使用规整网格、细分隔线或紧凑卡片。
- 核心状态、教练提示和关键行动使用不规则色块、错位叠层、贴纸标签或局部溢出图像。
- 每屏只允许一个强视觉焦点，其余模块降低饱和度和阴影，避免所有内容争夺注意力。
- 圆角分为小、中、大和自由形状四个层级；自由形状只用于品牌表达，不承载高密度文本。
- 阴影短而明确，并搭配墨色描边或色块关系；避免大面积柔焦光晕。

## 页面设计

### Welcome

首屏采用海报式排版：大标题、深色底部色块和能量伙伴主插画。价值主张被压缩为三条短句，并以小型贴纸图标呈现。后续资料填写继续使用现有三步状态，但表单改为更宽松的单主题版式。

### Check-in

保留现有字段与提交逻辑，将长表单改为逐题聚焦的分段体验。当前问题占据主视觉区，已完成答案收拢为上方进度条或状态胶囊。选择时提供颜色填充、轻微缩放和触觉反馈；最后一题完成后，能量伙伴从等待状态切换到准备完成状态。

### Today

首页变成杂志封面式仪表盘。顶部欢迎语与日期保持简洁；首屏主卡通过大号分数、能量伙伴和一句行动建议表达当天状态。能量曲线位于首屏下半部并具备绘制入场动画。峰值、低谷、营养和教练建议使用不同容器语法，形成有规则与无规则模块交替的节奏。

### Plan

计划页面以纵向能量轨道替代重复卡片。时间、任务和能量等级共用一条清晰时间轴；重点任务使用实色块，恢复与饮食使用浅色块。重新生成操作成为底部悬浮动作条，任务列表保留规整结构。

### Nutrition

营养页面采用食材拼贴式首屏。冰箱食材用可横向滚动的标签与生成装饰图组合，身体需求保持精确条形图。餐食建议使用错位图片、色块和短标签，但数值与准备时间仍遵循清晰网格。

## 交互与动效

- 页面进入：模块按 40–70ms 间隔错峰淡入并轻微上移，总时长不超过 450ms。
- 卡片按压：缩放到 0.98、阴影收紧，松开后平滑恢复。
- 能量分数：数字计数与曲线绘制同步完成。
- Check-in 选择：当前选项弹性填充，进度立即前进；减少动态效果设置开启时退化为无位移动画。
- Tab 切换：只动画图标状态和选中色块，不对整页做夸张转场。
- 角色状态：只在关键节点切换，避免持续循环动画干扰阅读。

## 组件与代码边界

- `theme/tokens.ts` 继续作为颜色、排版、间距、圆角、描边和动效时长的唯一来源。
- 基础组件拆分为规则容器、自由焦点容器、贴纸标签、悬浮动作条和动效包装器；页面只组合组件，不重复定义交互语法。
- 现有 `AppStateProvider`、`FixtureService`、领域类型和路由保持不变。
- 生成图片放入 `apps/app/assets/images/akeso/`，使用明确的状态命名，并保留透明背景版本。
- 图片加载失败时使用纯色色块和文字状态作为降级，不影响任务完成。

## 数据与错误状态

- 页面继续直接读取现有 `useAppState()` 数据，不引入新的持久化或网络依赖。
- Loading 使用角色或局部骨架屏表达，不使用全屏旋转器。
- Error 使用内联高对比提示与重试动作，不让错误提示改变页面整体高度过多。
- 空状态保留现有业务行动，例如先完成 Check-in，再生成计划。

## 可访问性与适配

- 正文和关键数值满足可读对比度，颜色不作为能量等级的唯一信息。
- 所有可点击区域至少 44×44pt，并保留可访问性标签。
- 支持 iOS、Android 和 Web；核心布局在 320–560pt 内容宽度内成立。
- 尊重系统减少动态效果设置；动效关闭后信息层级和状态反馈仍完整。

## 验证标准

- TypeScript 类型检查通过，现有业务数据与操作行为不变。
- Welcome → Check-in → Today → Plan → Nutrition 的核心链路可运行。
- 检查 loading、error、empty、high/moderate/low energy 等状态。
- 在窄屏手机与 Web 移动宽度下进行截图检查，确认无溢出、遮挡和 Tab 冲突。
- 视觉验收重点：每屏只有一个主焦点；规则与自由模块形成节奏；动效帮助理解状态而非装饰性堆叠。

## 非目标

- 不改动 API contract、能量算法、领域模型或后端接入方式。
- 不新增社交、账户、通知、扫描收据等产品功能。
- 不为了展示效果引入持续运行的复杂 3D 或重型动画依赖。
